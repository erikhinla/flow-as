# Hermes Escalation Policy

## Escalate immediately if a task touches
- production deployment
- secrets
- credentials
- DNS
- domain configuration
- database mutation
- rollback-sensitive infra changes
- security configuration
- payment logic
- destructive file operations without recovery path

## Escalate if the task would
- redefine canon
- alter brand roles
- introduce new naming
- reposition FLOW Agent OS above doctrine
- mix activation before readiness
- create a second source of truth

## Escalation output must include
- task summary
- why Hermes cannot continue safely
- affected surface
- proposed next actor
- rollback concern
- files touched so far
