# Execution Engine Overview

Authority: _OS/02-Canon/CANONICAL_SOURCE_OF_TRUTH.md (Section 9)

## What FLOW Agent OS Is

FLOW Agent OS is the execution engine.

It is the governed mechanism that routes and carries out work.

It is responsible for:

- governed agent execution
- task routing
- validation
- escalation
- rollback safety
- multi-agent coordination
- repeatable operational motion

## What FLOW Agent OS Is Not

FLOW Agent OS is not the whole thesis.
It is the execution engine that implements the orchestration system.

- Not the business doctrine (that is TransformBy10X)
- Not the infrastructure offer (that is BizBuilders)
- Not the activation layer (that is BizBot)
- Not the interface layer (that is AVA/AIVA)
- Not a replacement for canon

## Three Agents

| Agent | Risk Tier | Scope |
|---|---|---|
| OpenClaw | Alpha/Beta | Low-risk repo restructuring, content migration, artifact classification |
| Agent Zero | Gamma | High-risk deployment, secrets, infra config, rollback-required changes |
| Hermes | Operational | Bounded task execution, workflow dispatch, canon-governed deliverables |

## How It Fits

The execution engine sits within the canonical system architecture:

1. Context → 2. Governance → 3. Orchestration → **4. Execution** → 5. Memory → 6. Adaptation

FLOW Agent OS operates at layer 4. It does not define layers 1-3. It executes within them.
