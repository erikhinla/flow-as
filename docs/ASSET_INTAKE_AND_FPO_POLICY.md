# Asset Intake and FPO Policy

Status: active operating rule.

Purpose: prevent missing logos, photos, screenshots, or visual references from blocking FLOW Agent AS, TransformBy10X, BizBuilders AI, BizBot Marketing, AVA, or Prompting Circumstance work.

## Primary Rule

The founder must never become the bottleneck for creative progress.

If a required asset is missing, agents must continue with an FPO asset.

FPO = For Placement Only.

## Required Behavior

Agents must:

- continue page, layout, copy, and prototype work with an FPO asset when the final asset is missing
- label the asset slot clearly as FPO in internal artifacts
- record the needed replacement in the asset index
- avoid publishing FPO assets to production unless explicitly approved
- keep work moving while awaiting final logos, photos, screenshots, or product mockups

Agents must not:

- stop a site/page/task because a logo is missing
- wait for a founder photo before drafting layout
- block a prototype because a screenshot is unavailable
- invent final brand marks and treat them as approved
- publish placeholder/fake proof assets as real proof

## FPO Asset Types

Allowed FPO assets:

- neutral logo placeholder
- labeled image placeholder
- generated concept image clearly marked as draft
- simple shape/block composition
- screenshot placeholder frame
- product mockup placeholder
- diagram placeholder

Disallowed as final production assets:

- fake testimonials
- fake client screenshots
- fake analytics
- fake dashboard proof
- generated founder photos
- generated logos presented as official marks without approval

## Asset Drop Locations

Shared ecosystem assets:

- `/Users/guest1/Documents/GitHub/_ecosystem-assets/reference`

FLOW-specific references:

- `/Users/guest1/Documents/GitHub/FLOW-AGENT-AS/assets/reference`

## Asset Index Rule

Every visual task must maintain an asset index with:

- asset slot
- project/site
- page or surface
- current asset path
- FPO or final status
- replacement needed
- owner role
- approval status

## Notification Rule

If an asset is missing, the agent should notify the founder through the available operator channel, but must continue working with FPO.

Preferred internal notification text:

`FPO used: [asset slot]. Work continues. Final asset can replace it later.`

## Approval Gates

Human approval is required before:

- final logo adoption
- founder image publication
- public proof-of-work screenshots
- claims-bearing graphics
- paid product mockups
- production launch

Human approval is not required to:

- use FPO assets in drafts
- build layouts around placeholder slots
- generate internal mockups
- create wireframes
- create asset request lists

## FLOW Routing

Most asset intake and FPO decisions route to Beta.

Reputation-sensitive visual decisions route to Alpha.

Production launch, claims, legal/trademark, payment, or public proof assets route to Gamma if they create security, money, downtime, or legal exposure.
